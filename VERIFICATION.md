# 验证指南 - 完整命令执行链路

## 📋 验证步骤

### 第一步：启动服务

#### 方法一：使用启动脚本（推荐）

**macOS/Linux:**
```bash
chmod +x start.sh
./start.sh
```

**Windows:**
```cmd
start.bat
```

#### 方法二：手动启动

**终端 1 - 启动后端：**
```bash
cd server
npm run dev
```

**终端 2 - 启动前端：**
```bash
npm run dev
```

### 第二步：验证服务状态

#### 1. 检查后端健康状态

打开浏览器访问：http://localhost:3001/health

**预期结果：**
```json
{
  "status": "ok",
  "timestamp": "2026-03-13T07:46:36.838Z",
  "activeSessions": 0
}
```

#### 2. 检查前端页面

打开浏览器访问：http://localhost:3000

**预期结果：**
- 看到登录页面
- 显示"在线命令执行器"标题
- 有用户名和密码输入框

### 第三步：验证登录流程

#### 1. 使用演示账号登录

**输入：**
- 用户名: `admin`
- 密码: `admin123`

**预期结果：**
- 登录成功
- 自动跳转到终端页面
- 自动创建第一个终端会话
- 显示 "Connected to server" 消息

#### 2. 查看浏览器控制台

打开浏览器开发者工具（F12），查看 Console 和 Network 标签：

**预期看到：**
- WebSocket 连接成功（ws://localhost:3001）
- 认证消息发送
- 会话创建消息接收

### 第四步：验证命令执行

#### 测试命令 1：基础命令

**输入命令：**
```bash
ls -la
```

**预期结果：**
- 显示当前目录的文件列表
- 文件权限、大小、日期等信息
- ANSI 颜色正确显示

#### 测试命令 2：路径命令

**输入命令：**
```bash
pwd
```

**预期结果：**
- 显示当前工作路径
- 例如：`/Users/yourusername` 或 `/root`

#### 测试命令 3：用户信息

**输入命令：**
```bash
whoami
```

**预期结果：**
- 显示当前用户名
- 例如：`yourusername` 或 `root`

**输入命令：**
```bash
uname -a
```

**预期结果：**
- 显示系统详细信息

**输入命令：**
```bash

```

**预期结果：**
- 切换到 /tmp 目录

**输入命令：**

**预期结果：**

#### 1. 创建新会话

**操作：**

**预期结果：**
- 两个会话独立运行

- 点击标签上的 "X" 按钮

**预期结果：**

## 🐛 常见问题

### 问题 1：无法连接后端

**症状：** 登录时提示"登录失败，请检查网络连接"

**解决方案：**
1. 检查后端是否启动：访问 http://localhost:3001/health
2. 检查端口是否被占用：`lsof -i :3001`
3. 查看后端日志是否有错误

### 问题 2：WebSocket 连接失败

**症状：** 登录成功但终端无响应

**解决方案：**
1. 检查浏览器控制台是否有 WebSocket 错误
2. 确认后端 WebSocket 服务正常运行
3. 检查防火墙设置

### 问题 3：命令无输出

**症状：** 输入命令后没有反应

**解决方案：**
1. 检查 WebSocket 连接状态
2. 查看浏览器控制台是否有错误
3. 检查后端日志是否收到命令

### 问题 4：PTY 进程启动失败

**症状：** 后端日志显示 PTY 错误

**解决方案：**
1. 确认系统支持 PTY（Linux/macOS）
2. 检查 node-pty 是否正确安装
3. 尝试重新安装：`cd server && npm rebuild node-pty`
