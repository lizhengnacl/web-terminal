# 007-在线命令执行器 - 交互设计蓝图

---

## 一、产品概述

### 1.1 核心目标
构建一个现代Web风格的在线命令执行器，让开发者和运维工程师能够通过浏览器便捷地访问服务器终端，执行命令、管理文件，提供接近本地终端的流畅体验。

### 1.2 目标用户
**主要用户**：开发者、运维工程师

**用户画像**：
- 需要快速在远程服务器执行命令的开发者
- 管理多台服务器的运维工程师
- 需要轻量级服务器管理工具的技术人员
- 团队协作中需要共享服务器访问的成员

### 1.3 核心使用场景
| 场景 | 描述 | 核心需求 |
|------|------|----------|
| 远程调试 | 在远程服务器上快速执行命令调试代码 | 实时输出、多会话 |
| 服务器运维 | 日常服务器管理和维护 | 文件管理、快捷命令 |
| 命令复用 | 查找和复用历史执行过的命令 | 历史记录、搜索 |
| 多任务处理 | 同时执行多个独立任务 | 多个终端标签页 |

### 1.4 关键任务识别
1. **登录认证** - 通过密码验证身份
2. **终端交互** - 输入命令、查看实时输出
3. **会话管理** - 新建、切换、关闭终端标签
4. **文件操作** - 上传、下载文件
5. **历史查询** - 查看和搜索命令执行历史

## 二、整体页面流

### 2.1 页面结构
本产品采用**多页面应用**架构，主要包含以下页面：

```
┌─────────────────────────────────────────────────────────┐
│                    登录页面 (Login)                       │
│                     (入口页面)                            │
└──────────────────────┬────────────────────────────────────┘
                       │ 登录成功
                       ↓
┌─────────────────────────────────────────────────────────┐
│                  主应用页面 (Main App)                    │
│  ┌─────────────────────────────────────────────────────┐ │
│  │         顶部导航栏 (Global Header)                  │ │
│  ├─────────────────────────────────────────────────────┤ │
│  │                                                     │ │
│  │  ┌──────────┐  ┌─────────────────────────────────┐ │ │
│  │  │ 侧边栏   │  │      主内容区域                  │ │ │
│  │  │ (Sidebar)│  │   (根据当前导航切换)             │ │ │
│  │  │          │  │                                 │ │ │
│  │  └──────────┘  └─────────────────────────────────┘ │ │
│  └─────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘

主内容区域可切换为：
  - 终端页面 (Terminal) - 默认
  - 文件管理页面 (Files)
  - 历史记录页面 (History)
```

### 2.2 页面流转关系

```
用户访问
    │
    ├─→ 未登录 ──→ 登录页面 ──→ 输入密码验证 ──→ [成功] 主应用页面
    │                                      │
    │                                      └─→ [失败] 登录页面 (显示错误)
    │
    └─→ 已登录 ──→ 主应用页面
                  │
                  ├─→ 默认显示：终端页面
                  │
                  ├─→ 点击"文件" → 文件管理页面
                  │      │
                  │      └─→ 返回"终端" → 终端页面
                  │
                  ├─→ 点击"历史" → 历史记录页面
                  │      │
                  │      └─→ 返回"终端" → 终端页面
                  │
                  └─→ 点击"退出" → 清除登录状态 → 登录页面
```

## 四、核心交互流程

### 流程1：登录并执行命令（核心任务闭环）

**流程目标**：用户登录后在终端执行命令并查看输出

**参与模块**：登录页面 → 顶部导航栏 → 终端标签栏 → 终端输出区 → 命令输入区

**详细步骤**：

```
1. 用户访问应用
   └─→ 未登录：显示登录页面

2. 用户输入密码
   └─→ 点击登录按钮
   └─→ 按钮显示加载态

3. 验证通过
   └─→ 跳转到主应用页面
   └─→ 默认显示终端页面
   └─→ 自动创建第一个终端标签

4. 用户在命令输入区输入命令
   └─→ 实时显示输入内容
   └─→ 按 Tab 键可触发自动补全

5. 用户按 Enter 键执行命令
   └─→ 命令添加到输出区
   └─→ 输入区清空
   └─→ 输出区实时显示命令输出

6. 命令执行完成
   └─→ 显示命令提示符
   └─→ 等待下一个命令

7. 用户查看输出
   └─→ 如需滚动查看历史，关闭自动滚动
   └─→ 可复制输出内容
```

**异常分支**：
- 登录失败：显示错误提示，用户可重新输入
- 命令执行失败：输出区显示错误信息，命令退出码
- 网络中断：显示断线提示，自动尝试重连

---

### 流程2：使用多个终端标签（多会话任务）

**流程目标**：用户创建和管理多个终端会话

**参与模块**：终端标签栏 → 终端输出区 → 命令输入区

**详细步骤**：

```
1. 用户在终端页面
   └─→ 已有一个终端标签在运行

2. 用户点击 "+" 按钮
   └─→ 创建新的终端标签"终端2"
   └─→ 自动切换到新标签

3. 用户在新终端执行命令
   └─→ 独立的会话环境

4. 用户点击"终端1"标签
   └─→ 切换回第一个终端
   └─→ 恢复之前的会话状态

5. 用户点击标签上的 "×"
   └─→ 如是最后一个标签：弹出确认"关闭最后一个终端？"
   └─→ 用户确认：关闭标签
   └─→ 自动切换到上一个标签
```

### 流程3：上传文件到服务器（文件操作任务）

**流程目标**：用户通过Web界面上传文件到服务器

**参与模块**：顶部导航栏 → 侧边栏 → 文件管理页面

**详细步骤**：

```
1. 用户在终端页面
   └─→ 点击顶部导航"文件"
   └─→ 或点击侧边栏"上传文件"按钮

2. 切换到文件管理页面
   └─→ 显示目录树
   └─→ 显示上传区域

3. 用户选择文件
   └─→ 方式1：拖拽文件到上传区域
   └─→ 方式2：点击"选择文件"按钮，在对话框中选择

4. 用户确认上传
   └─→ 显示上传进度
   └─→ 上传完成显示成功提示

5. 用户查看文件
   └─→ 目录树刷新，显示新上传的文件
   └─→ 可在终端中验证文件存在
```

### 流程4：查看和复用历史命令（历史任务）

**流程目标**：用户查找历史命令并复用

**参与模块**：顶部导航栏 → 历史记录页面

**详细步骤**：

```
1. 用户点击顶部导航"历史"
   └─→ 切换到历史记录页面
   └─→ 显示命令执行历史列表

2. 用户搜索命令
   └─→ 在搜索框输入关键词
   └─→ 列表实时过滤

3. 用户找到目标命令
   └─→ 点击历史项展开
   └─→ 查看详细输出

4. 用户复用命令
   └─→ 点击"复用"按钮
   └─→ 自动跳转到终端页面
   └─→ 命令自动填入输入框
   └─→ 用户可修改后执行
```

## 五、响应式适配规则

### 桌面端（≥1024px）
- 完整三栏布局：顶部导航 + 左侧边栏 + 主内容区
- 侧边栏宽度固定 200px
- 所有功能完整展示

### 平板端（768px - 1023px）
- 侧边栏可折叠，默认展开
- 侧边栏宽度可调整
- 标签栏支持横向滚动

### 移动端（<768px）
- 侧边栏隐藏，通过汉堡菜单展开
- 终端输入框固定在底部
- 标签栏改为底部导航或下拉选择

## 六、边缘场景处理

| 场景 | 触发条件 | 处理方式 |
|------|----------|----------|
| 长耗时命令 | 命令执行超过30秒 | 显示"命令仍在执行中"提示，不阻塞界面 |
| 大量输出 | 输出超过10000行 | 自动截断早期输出，保留最新内容 |
| 网络中断 | WebSocket连接断开 | 显示断线提示，自动尝试重连 |
| 会话过期 | 登录超时 | 弹出提示，跳转到登录页面 |
| 命令执行失败 | 命令返回非零退出码 | 输出区红色高亮显示错误信息 |
| ANSI颜色码 | 输出包含颜色控制字符 | 正确解析并显示彩色文本 |
| 并发访问 | 多人同时使用 | 会话隔离，互不干扰 |
